# 나만의 퀴즈 게임 - 개발 로드맵

---

## 개요

프로그래밍/IT 주제의 터미널 퀴즈 게임을 처음부터 끝까지 구현하는 로드맵이다.

- **퀴즈 주제**: 프로그래밍/IT (Python, 컴퓨터 과학, 개발 상식)
- **파일 구조**: 다중 파일 (`quiz.py`, `quiz_game.py`, `main.py`)
- **데이터 저장**: `state.json` (UTF-8)
- **개발 환경**: Python 3.10 이상, 외부 라이브러리 미사용 (표준 라이브러리만 허용)

### 목표 파일 구조

```
E1-2/
├── main.py              # 프로그램 진입점
├── quiz.py              # Quiz 클래스
├── quiz_game.py         # QuizGame 클래스
├── state.json           # 런타임 데이터 (.gitignore 포함)
├── .gitignore
├── README.md
└── docs/
    ├── MISSION.md
    ├── ROADMAP.md
    └── screenshots/
        ├── menu.png
        ├── play.png
        ├── add_quiz.png
        └── score.png
```

> **과제 목표 3-4 — Git 기초**
>
> Git은 코드의 변경 이력을 관리하는 **버전 관리 시스템**이다. 왜 필요한가?
>
> - 언제, 무엇을, 왜 바꿨는지 기록이 남는다
> - 실수했을 때 이전 상태로 되돌릴 수 있다

>
> | 명령어 | 역할 |
> |---|---|

> | `git commit` | 스테이징된 변경사항을 하나의 기록(커밋)으로 저장한다 |

### 학습 목표

> **과제 목표 3-2 — 클래스와 객체**
>
> 클래스는 관련된 데이터와 동작을 하나로 묶는 설계도이다.
>
> - **왜 사용하는가?** 퀴즈 한 문제에는 "문제 텍스트", "선택지 목록", "정답 번호"가 항상 함께 다닌다. 이것들을 각각 별도의 변수로 관리하면 데이터가 흩어지고 실수가 생기기 쉽다. 클래스로 묶으면 "퀴즈 하나 = 객체 하나"로 깔끔하게 관리할 수 있다.
> - **`__init__` 메서드**: 객체가 생성될 때 자동으로 호출되어 초기 데이터를 세팅한다.
> - **`self`**: "이 객체 자신"을 가리킨다. 객체 내부의 데이터에 접근하거나 다른 메서드를 호출할 때 사용한다.
> - **속성(attribute)**: 객체가 가진 데이터 (예: 문제 텍스트, 선택지)
> - **메서드(method)**: 객체가 할 수 있는 동작 (예: 퀴즈 출력, 정답 확인)

> **과제 목표 3-1 — Python 기초 (변수와 자료형)**
>
> 변수는 데이터에 이름을 붙여서 저장하는 것이다. 퀴즈 게임에서 사용하는 주요 자료형:
>
> | 자료형 | 설명 | 퀴즈 게임에서의 예시 |
> |---|---|---|
> | `str` | 문자열 | 문제 텍스트, 선택지 |
> | `int` | 정수 | 정답 번호, 점수 |
> | `bool` | 참/거짓 | 정답 여부 |
> | `list` | 순서가 있는 목록 | 선택지 4개의 목록, 퀴즈 목록 |
> | `dict` | 키-값 쌍 | JSON 저장용 딕셔너리 |

### 학습 목표

> **과제 목표 3-3 — 파일 입출력**
>
> 프로그램이 종료되면 메모리의 데이터는 사라진다. 파일에 저장해두면 다음 실행 시 불러올 수 있다.
>
> - **파일 열기/읽기/쓰기**: `open()` 함수로 파일을 열고, 읽거나 쓴 뒤 닫는 것이 기본 흐름이다.
> - **JSON**: 사람이 읽을 수 있는 텍스트 기반 데이터 형식이다. Python의 `dict`/`list`와 1:1 대응되어 변환이 쉽고, 표준 라이브러리 `json` 모듈만으로 처리할 수 있다.
> - **`try`/`except`**: 파일이 없거나 손상되었을 때 프로그램이 멈추지 않도록 오류를 잡아서 처리한다. "일단 시도하고, 실패하면 대안을 실행"하는 구조이다.

> **과제 목표 3-1 — Python 기초 (조건문, 반복문, 함수)**
>
> - **`if`/`elif`/`else`**: 조건에 따라 다른 동작을 수행한다. 메뉴에서 사용자가 입력한 번호에 따라 다른 기능을 실행할 때 사용한다.
> - **`while`**: 조건이 참인 동안 반복한다. 메뉴를 "종료를 선택할 때까지" 계속 보여주는 데 적합하다.
> - **함수**: 코드를 이름 붙여 묶어놓은 것이다. 매개변수로 입력을 받고, 반환값으로 결과를 돌려준다. 기능별로 함수를 나누면 코드가 읽기 쉽고 재사용할 수 있다.

### 학습 목표

> **과제 목표 3-4 — Git 기초 (브랜치와 병합)**
>
> 브랜치는 코드의 독립적인 작업 공간이다. `main` 브랜치를 건드리지 않고 새로운 기능을 개발할 수 있다.
>
> | 명령어 | 역할 |
> |---|---|
> | `git checkout -b feature/quiz-play` | 새 브랜치를 만들면서 이동한다 |
> | `git checkout main` | `main` 브랜치로 돌아간다 |
> | `git merge feature/quiz-play` | 브랜치의 작업을 `main`에 합친다 |
>
> 기능 단위로 브랜치를 나눠 작업하면, 미완성 코드가 `main`에 섞이지 않고, 작업 기록이 깔끔하게 남는다. 이 방식은 이후 팀 협업(PR, 코드 리뷰)의 기반이 된다.

> **과제 목표 3-1 — Python 기초 (반복문 선택, 함수 활용)**
>
> - **`for` vs `while`**: `for`는 "정해진 횟수/목록만큼" 반복할 때, `while`은 "조건이 만족될 때까지" 반복할 때 사용한다. 퀴즈 목록을 순회하며 출제할 때는 `for`, 올바른 입력이 들어올 때까지 반복할 때는 `while`이 적절하다.
> - **함수의 매개변수와 반환값**: 함수에 입력을 넘기고(매개변수) 결과를 받아오는(반환값) 구조로 기능을 분리하면, 각 부분을 독립적으로 이해하고 테스트할 수 있다.

> **과제 목표 3-1 — Python 기초 (표준 라이브러리 활용)**
>
> Python은 외부 라이브러리 없이도 많은 기능을 제공하는 표준 라이브러리가 풍부하다.
>
> - **`random` 모듈**: 리스트를 섞거나(`shuffle`), 일부를 랜덤으로 뽑는(`sample`) 기능을 제공한다. 퀴즈 순서를 무작위로 만들 때 사용한다.
> - **`datetime` 모듈**: 현재 날짜와 시간을 다루는 기능을 제공한다. 게임 기록에 타임스탬프를 남길 때 사용한다.
> - **리스트 슬라이싱**: `list[:n]`처럼 리스트의 일부만 잘라낼 수 있다. 선택한 문제 수만큼만 출제할 때 활용한다.

### 학습 목표

> **과제 목표 3-4 — Git 기초 (clone, pull)**
>
> | 명령어 | 역할 |
> |---|---|
> | `git clone <URL>` | 원격 저장소 전체를 로컬에 복제한다. 새 컴퓨터에서 작업을 시작하거나 다른 사람의 코드를 가져올 때 사용한다. |
> | `git pull` | 원격 저장소의 변경사항을 로컬로 가져와 병합한다. 다른 곳에서 push한 내용을 동기화할 때 사용한다. |
>
> 이 실습을 통해 `init` → `add` → `commit` → `push` → `pull` → `checkout` → `clone` 7종 Git 명령어를 모두 사용하게 된다.
